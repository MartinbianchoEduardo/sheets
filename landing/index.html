Arquitetura

Frontend sem segredo. Worker como único trust boundary.

A página estática é totalmente pública — qualquer pessoa pode ver o código-fonte — e mesmo assim não tem nenhuma credencial explorável. Toda chamada de backend exige uma assinatura WebAuthn da passkey atrelada ao Secure Enclave do dispositivo.

O Worker em Cloudflare faz as cerimônias WebAuthn, emite JWTs HS256 de 15 minutos e é a única coisa que enxerga o D1.

  • 01
    Passkey + Secure Enclave Face ID assina um challenge. Chave privada nunca sai do hardware.
  • 02
    Challenge stateless HMAC-signed token passado pelo cliente. Nada de KV, nada de replay por estado stale.
  • 03
    JWT curto Sessão de 15 minutos, refresh proativo antes do vencimento. Tab aberta nunca cai.
  • 04
    D1 isolado Banco SQLite só acessível via binding do Worker. Frontend não toca no dado.

Stack

Pequeno, direto, sem inchaço.

Cada peça escolhida pra resolver uma coisa específica — e nada mais.

Frontend
Preact + Vite
JSX, plain JS, sem TypeScript. State com @preact/signals.
Auth
WebAuthn + JWT
SimpleWebAuthn no Worker. HS256 com refresh proativo.
Backend
Cloudflare Workers
ES modules + Web Crypto. Roteamento próprio, sem framework.
Data
D1 · SQLite
Schema migrado de Apps Script. Binding direto, sem ORM.
Data fetching
TanStack Query
Stale-while-revalidate, um hook por endpoint.
Hosting
Cloudflare Pages
Estático, edge-cached, deploy via wrangler.
